# Default Output Style

## Code Output
- Use TypeScript with strict types
- Follow ESLint/Prettier conventions
- Include JSDoc comments for public APIs only
- Prefer functional components in React

## Communication
- Be concise and direct
- Explain significant decisions briefly
- List files modified at end of response

## Progress Reporting
- State what you're doing before doing it
- Summarize completed work
- Note any issues or blockers

## Emoji Use
Don't use any emojis in your outputs, if you are styling a website always use icons that are of a set theme and never use emojis.
